# Bear UI — Articles for dev.to, Medium, LinkedIn

This folder contains markdown articles for **Bear UI v1.1.4** to publish on:

- **dev.to** — `bear-ui-1.1.4-devto.md`
- **Medium** — `bear-ui-1.1.4-medium.md`
- **LinkedIn** — `bear-ui-1.1.4-linkedin.md`

## Screenshots to add

Place your images in the `screenshots/` folder. The articles reference these files (add them and the embeds will work):

| File | Description | Suggested capture |
|------|-------------|-------------------|
| `portal-hero.png` | Portal landing or component list | Full-width view of Bear portal home or Introduction page |
| `hovercard-loc-badge.png` | HoverCard page with green LOC badge | Component page title: "HoverCard" + green +95 badge (dark theme) |
| `components-dark.png` | Dark theme component examples | 2–3 components (e.g. Button, Card, Popconfirm) in dark mode |
| `components-light.png` | Light theme component examples | Same components in light mode |
| `new-components-grid.png` | New v1.1.4 components | Grid or list of new components (Result, RingProgress, CheckboxCard, etc.) |
| `anchor-scroll-spy.png` | Anchor / Table of Contents | Anchor component with scroll-spy nav visible |
| `installation-code.png` | Install + import snippet | Terminal or editor showing `npm install @forgedevstack/bear` and first import |

### Publishing

- **dev.to** — Paste the content from `bear-ui-1.1.4-devto.md`. Upload each image via the editor (drag & drop or image button) and replace `screenshots/xxx.png` with the URL dev.to gives you. Add tags (e.g. `react`, `typescript`, `tailwindcss`, `ui`).
- **Medium** — Same as dev.to: paste from `bear-ui-1.1.4-medium.md`, upload images in the Medium editor, and replace image paths with the uploaded image URLs.
- **LinkedIn** — Paste `bear-ui-1.1.4-linkedin.md` as a post. You can set one link (e.g. to bearui.com or the dev.to article) and optionally attach a single hero image (e.g. `portal-hero.png` or `hovercard-loc-badge.png`). No need to embed all screenshots in the post body.
